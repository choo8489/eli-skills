# 로그 분석기

대상: $ARGUMENTS

## 분석 프로세스

### 1. 로그 형식 감지
- JSON 구조화 로그
- 평문 로그 (timestamp + level + message)
- 멀티라인 스택 트레이스
- 커스텀 형식 (자동 파싱 시도)

### 2. 에러 분석

#### 에러 그룹핑
- 동일한 에러를 스택 트레이스 기준으로 그룹화
- 발생 빈도 카운트
- 첫 발생 / 마지막 발생 시각
- 영향받는 사용자/요청 수

#### 에러 패턴
- 반복적 에러 (cron job, retry 루프)
- 폭발적 증가 (cascading failure)
- 주기적 패턴 (특정 시간대)
- 새로 등장한 에러

### 3. 성능 분석
- 응답 시간 분포 (p50, p95, p99)
- 느린 요청 식별 (slow query, timeout)
- 처리량 변화 (requests/sec)
- 메모리/CPU 관련 로그

### 4. 타임라인 분석
- 장애 발생 전후 이벤트 순서
- 연관된 서비스/모듈 간 이벤트 상관관계
- 배포/변경 이벤트와의 시간적 연관성

### 5. 이상 징후
- 평소와 다른 패턴
- 갑작스러운 로그 볼륨 변화
- 새로운 에러 타입 등장
- 특정 사용자/IP에서 집중된 에러

## 출력 형식

```
## 로그 분석 결과

**기간**: [시작] ~ [종료]
**총 라인**: N줄
**에러 비율**: X%

### 에러 요약 (빈도순)
| # | 에러 타입 | 횟수 | 첫 발생 | 마지막 발생 |
|---|----------|------|---------|------------|
| 1 | NullPointerException at UserService:45 | 342 | 10:23 | 14:51 |

### 타임라인
[시간순 주요 이벤트]

### 근본 원인 추정
1. [가능성 높은 원인] — 근거: [로그 증거]

### 권장 조치
1. [즉시 조치]
2. [모니터링 강화]
3. [근본 해결]
```

## 규칙
- 큰 로그는 패턴/샘플링 기반으로 분석
- 민감 정보 (토큰, 비밀번호) 가 로그에 있으면 경고
- 추측 대신 로그 증거를 기반으로 분석
- 상관관계와 인과관계를 구분하여 설명
