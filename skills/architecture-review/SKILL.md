# 아키텍처 리뷰

대상: $ARGUMENTS

## 리뷰 관점

### 1. 확장성 (Scalability)
- 수평/수직 확장 가능성
- 병목 지점 식별 (DB, 네트워크, 컴퓨팅)
- 캐싱 전략 적절성
- 비동기 처리가 필요한 구간

### 2. 신뢰성 (Reliability)
- 단일 장애 지점 (SPOF) 식별
- 장애 전파 방지 (Circuit Breaker, Bulkhead)
- 데이터 일관성 보장 방법
- 재시도/폴백 전략
- 복구 시간 목표 (RTO/RPO)

### 3. 보안 (Security)
- 인증/인가 아키텍처
- 네트워크 격리 (VPC, 서브넷)
- 데이터 암호화 (전송 중/저장 중)
- 시크릿 관리 방법

### 4. 비용 (Cost)
- 리소스 사용 효율성
- 오버프로비저닝 여부
- 서버리스 vs 상시 가동 적절성
- 데이터 전송 비용

### 5. 유지보수성 (Maintainability)
- 서비스/모듈 간 결합도
- 배포 독립성
- 모니터링/로깅 체계
- 문서화 수준

```
### 위험 요소

   → 권장: ...

### 개선 로드맵
- 단기 (1-2주): ...
- 중기 (1-3개월): ...
- 장기 (3개월+): ...
```

## 규칙
- 현재 규모와 팀 상황을 고려한 현실적 제안
- 과도한 엔지니어링 경고 (YAGNI)
- 트레이드오프를 명확히 설명
- 다이어그램으로 시각화
