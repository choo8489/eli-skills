# 코드 복잡도 분석기

대상: $ARGUMENTS

## 측정 지표

### 1. 순환 복잡도 (Cyclomatic Complexity)
분기 경로 수를 측정합니다.
- `if`, `else if`, `case`, `&&`, `||`, `?.`, `??` 각각 +1
- **1-5**: 단순 (양호)
- **6-10**: 보통 (주의)
- **11-20**: 복잡 (리팩토링 권장)
- **21+**: 매우 복잡 (리팩토링 필수)

### 2. 인지 복잡도 (Cognitive Complexity)
코드를 이해하는 데 필요한 정신적 부하를 측정합니다.
- 중첩 구조마다 가중치 증가
- `break`, `continue`, `goto` 추가 가중치
- 재귀 호출 추가 가중치

### 3. 함수/메서드 길이
- **1-15줄**: 이상적
- **16-30줄**: 허용
- **31-60줄**: 분리 고려
- **60줄+**: 분리 필수

### 4. 중첩 깊이 (Nesting Depth)
- **1-2단계**: 양호
- **3단계**: 주의
- **4단계+**: Early Return 패턴 적용 필요

### 5. 파라미터 개수
- **0-3개**: 이상적
- **4-5개**: 객체로 묶기 고려
- **6개+**: 반드시 리팩토링

## 분석 프로세스

### 1. 파일/디렉토리 스캔
- 모든 함수/메서드 식별
- 각 지표 계산
- 임계값 초과 항목 표시

### 2. 핫스팟 식별
- 복잡도 높은 파일 TOP 10
- 변경 빈도가 높은 복잡 코드 (git log 참고)
- 테스트 커버리지가 낮은 복잡 코드

### 3. SOLID 위반 검사
- **S**: 하나의 클래스/함수가 너무 많은 일을 하는지
- **O**: 수정 없이 확장 가능한지
- **D**: 구체 클래스에 직접 의존하는지

## 출력 형식

```
## 복잡도 분석 결과

**전체 점수**: 7.2/10 (양호)
**분석 파일**: N개 / **함수**: M개

### 복잡도 TOP 10 (리팩토링 우선순위)
| 순위 | 파일:함수 | 순환복잡도 | 인지복잡도 | 줄수 | 중첩 |
|------|-----------|-----------|-----------|------|------|
| 1    | auth.ts:validateToken | 23 | 31 | 85 | 5 |

### 위험 영역
🔴 즉시 리팩토링: [목록]
🟡 계획적 개선: [목록]
🟢 양호: [비율]%

### 리팩토링 제안
1. [함수] → [구체적 개선 방법]
```

## 규칙
- 숫자만 나열하지 말고 실행 가능한 개선안 제시
- 테스트가 있는 코드를 우선 리팩토링 대상으로
- 복잡도가 높아도 변경 빈도가 낮으면 우선순위 낮춤
- 팀의 맥락을 고려한 현실적 기준 적용
