# 코드베이스 → 학습 자료

대상: $ARGUMENTS

## 프로세스

### 1. 코드베이스 분석
- 프로젝트 구조 파악 (디렉토리, 주요 파일)
- 진입점 (entry point) 식별
- 핵심 모듈과 의존 관계 매핑
- 데이터 흐름 추적 (요청 → 처리 → 응답)

### 2. 학습 경로 설계

#### 레벨 구조
```
Level 0: 프로젝트 개요 (전체 그림)
Level 1: 핵심 개념 (도메인 모델, 아키텍처)
Level 2: 주요 흐름 (사용자 시나리오별 코드 추적)
Level 3: 심화 (내부 구현, 최적화, 패턴)
Level 4: 실습 (직접 수정해보기)
```

#### 각 레벨에 포함할 것
- 학습 목표 (이 챕터를 마치면 알 수 있는 것)
- 핵심 파일 목록 (읽어야 할 순서)
- 개념 설명 (비유와 다이어그램 포함)
- 코드 워크스루 (주요 코드 블록 설명)
- 퀴즈/확인 질문
- 실습 과제

### 3. 콘텐츠 형식

#### 아키텍처 맵
```
[ASCII 다이어그램]
- 모듈 간 관계
- 데이터 흐름
- 외부 연동
```

#### 코드 워크스루
```
파일: src/auth/login.ts

1. [라인 10-15] 요청 파싱
   → 사용자가 보낸 이메일/비밀번호를 추출

2. [라인 17-25] DB 조회
   → 이메일로 사용자 검색, 없으면 에러

3. [라인 27-35] 비밀번호 검증
   → bcrypt로 해시 비교
   💡 평문 비교가 아닌 이유: [설명]
```

#### 용어 사전
- 프로젝트에서 사용하는 도메인 용어 정리
- 약어 풀이

## 출력 형식

```
## [프로젝트 이름] 학습 가이드

### 🗺️ 프로젝트 맵
[아키텍처 다이어그램]

### 📚 학습 경로
- Chapter 1: [제목] — [소요시간]
- Chapter 2: [제목] — [소요시간]
- ...

### Chapter 1: [제목]
**목표**: [이 챕터를 마치면...]
**핵심 파일**: [파일 목록]

[내용]

**확인 질문**:
1. ...

**실습**:
- ...
```

## 규칙
- 코드를 처음 보는 사람의 관점으로 작성
- 파일을 읽어야 하는 순서를 명확히 제시
- 모든 전문 용어는 첫 등장 시 설명
- "왜 이렇게 만들었는지"를 코드 설명에 포함
- 실습 과제는 실제로 수행 가능한 것으로
