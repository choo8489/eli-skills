# 정규식 빌더 & 해석기

입력: $ARGUMENTS

## 모드 자동 감지

### 모드 A: 설명 → 정규식 생성
자연어 설명을 받아 정규식을 생성합니다.

### 모드 B: 정규식 → 해석
기존 정규식을 받아 구조를 분석하고 설명합니다.

## 생성 프로세스

### 1. 요구사항 파악
- 매칭할 대상 (이메일, URL, 전화번호, 커스텀 패턴 등)
- 허용/불허 문자
- 캡처 그룹 필요 여부
- 대소문자 구분, 멀티라인 등 플래그

### 2. 정규식 작성
- 단계적으로 구성요소 조합
- 가독성을 위해 주석 포함 (verbose 모드)
- 성능 고려 (catastrophic backtracking 방지)

### 3. 테스트 케이스
- 매칭되어야 할 예시 (최소 3개)
- 매칭되지 않아야 할 예시 (최소 3개)
- 엣지 케이스

## 출력 형식

```
## 정규식 결과

**패턴**: `/pattern/flags`
**언어별 사용법**: JS, Python, Go 등

### 구조 분석
/
  ^                 # 문자열 시작
  (?:pattern)       # 그룹 설명
  [a-z]             # 문자 클래스 설명
  $                 # 문자열 끝
/flags

### 테스트
✅ 매칭: "example1", "example2"
❌ 불일치: "counter1", "counter2"

### 주의사항
- [엣지 케이스나 한계점]
```

## 해석 프로세스

### 1. 토큰 분해
각 정규식 토큰을 한글로 설명:
- `^` → 문자열의 시작
- `\d{3}` → 숫자 3자리
- `(?=...)` → 긍정 전방탐색

### 2. 전체 동작 요약
- 이 정규식이 하는 일을 한 문장으로
- 매칭/불일치 예시 제시

### 3. 개선 제안
- 성능 최적화 가능성
- 엣지 케이스 누락
- 더 읽기 쉬운 대안

## 규칙
- 언어별 정규식 차이 명시 (JS vs Python vs Go)
- Catastrophic backtracking 패턴 경고
- 가능하면 named group 사용 권장
- 너무 복잡하면 여러 단계로 분리 제안
