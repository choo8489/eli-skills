# 성능 프로파일링

대상: $ARGUMENTS

## 분석 영역

### 1. 런타임 성능
- 시간 복잡도 분석 (Big-O)
- 핫 패스 (자주 실행되는 코드) 식별
- 불필요한 반복/재계산
- 동기 블로킹 작업
- N+1 쿼리 패턴

### 2. 메모리
- 메모리 누수 가능성 (이벤트 리스너, 클로저, 캐시)
- 대량 객체 생성/GC 압박
- 버퍼/스트림 미사용 (대용량 데이터)
- 순환 참조

### 3. 네트워크/IO
- 병렬 가능한 순차 요청
- 불필요한 API 호출
- 페이로드 크기 최적화
- 캐싱 미적용

### 4. 프론트엔드 (해당 시)
- 번들 사이즈 분석
- 불필요한 리렌더링
- 이미지/에셋 최적화
- 코드 스플리팅 기회
- Web Vitals (LCP, FID, CLS)

### 5. 데이터베이스 (해당 시)
- 슬로우 쿼리 식별
- 인덱스 누락
- 풀 테이블 스캔
- 커넥션 풀 설정

## 출력 형식

```
## 성능 분석 결과

**병목 요약**: [가장 큰 병목 한줄 설명]

### 병목 지점 (영향도 순)
1. 🔴 [파일:라인] - 설명
   - 현재: O(n²) / ~500ms
   - 개선: O(n log n) / ~50ms
   - 방법: [구체적 수정 방법]

2. 🟡 [파일:라인] - 설명
   ...

### 최적화 권장 순서
1. [가장 효과 큰 것부터]

### 벤치마크 제안
[성능 측정을 위한 코드/명령어]
```

## 규칙
- 추측이 아닌 코드 기반 분석
- 최적화 전 측정 가능한 기준 제시
- 가독성을 해치지 않는 범위에서 최적화
- 조기 최적화 경고 (실제로 문제가 되는 부분만)
