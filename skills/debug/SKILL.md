# 디버깅 도우미

문제: $ARGUMENTS

## 디버깅 프로세스

### 1. 문제 파악
- 에러 메시지 정확히 읽기
- 스택 트레이스에서 원인 위치 파악
- 재현 조건 확인

### 2. 원인 분석
- 관련 코드를 읽고 실행 흐름 추적
- 최근 변경사항 확인 (git log, git diff)
- 의존성 버전 호환성 확인
- 환경 차이 확인 (dev vs prod)

### 3. 가설 수립 및 검증
- 가능한 원인들을 가능성 높은 순으로 나열
- 각 가설을 검증할 수 있는 방법 제시
- 최소한의 변경으로 테스트

### 4. 해결 방안 제시
- 근본 원인(root cause) 해결 방법
- 임시 해결책(workaround)이 필요한 경우 함께 제시
- 재발 방지를 위한 제안

## 출력 형식

```
## 디버깅 결과

**에러**: [에러 메시지 요약]
**원인**: [근본 원인 한줄 설명]

### 분석
[상세 분석 내용]

### 해결 방법
1. [구체적인 수정 방법]

### 재발 방지
- [테스트 추가, 타입 강화 등]
```

## 규칙
- 추측하지 말고 코드를 읽어서 확인
- 가장 간단한 원인부터 확인 (오캄의 면도날)
- 해결 후 사이드이펙트 확인 필요
