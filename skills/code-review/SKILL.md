# 코드 리뷰

대상: $ARGUMENTS

## 리뷰 체크리스트

다음 항목들을 순서대로 검토하세요:

### 1. 버그 및 논리 오류
- 오프바이원(off-by-one) 에러
- null/undefined 처리 누락
- 경쟁 조건(race condition)
- 잘못된 타입 사용

### 2. 보안
- 입력값 검증 여부
- SQL 인젝션, XSS 가능성
- 민감한 정보 노출 (API 키, 비밀번호 등)
- 인증/인가 처리

### 3. 성능
- 불필요한 반복 / 중복 연산
- N+1 쿼리 문제
- 메모리 누수 가능성
- 적절한 자료구조 사용 여부

### 4. 가독성 및 유지보수성
- 네이밍 컨벤션 일관성
- 함수/클래스 크기 적정성
- 중복 코드 (DRY 원칙)
- 복잡도 (인덴트 깊이, 조건문 중첩)

### 5. 테스트
- 핵심 로직의 테스트 커버리지
- 엣지 케이스 테스트 여부
- 테스트 가독성

## 출력 형식

```
## 리뷰 결과

**심각도**: 🔴 Critical / 🟡 Warning / 🟢 Good

### 발견된 이슈
1. [심각도] 파일:라인 - 설명
   → 제안: ...

### 잘한 점
- ...

### 개선 제안
- ...
```

## 규칙
- 주관적 스타일 의견은 최소화하고 실질적 문제에 집중
- 각 이슈에 구체적인 수정 방법을 제안
- 칭찬할 부분도 반드시 포함
